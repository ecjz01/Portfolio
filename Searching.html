Searching

Searching, like the act of sorting is when you go through an entire list, however unlike sorting you are only looking for a specific item in the list and not trying to rearrange the list. The search method I will go over here will be binary search. Unfortunately binary search will only work on sorted lists. This is because the way binary search works, is that it checks if the middle of the list is number it is looking for. Then if it isn't it splits the list in half and then either takes the greater of smaller half, if the item you are searching for is higher or lower than the middle.

In the example above you can see that it first selects 14 as the middle, because it is in the 5th position because that is the middle of the 9 total positions. It sees that 14 is not 47 and then selects 45 as it is the middle of the remaining options. Then it goes to 47, because it sees that 45 isn't 47 and 47 is the remaining middle of the remaining elements. It then sees that it has found the item in the list.

The average complexity of binary search is O(log n), which is also the worst case scenario. This is because the list will go through at most log n iterations, because it is constantly divided into half. The best case scenario however is O(1), because th best case is when it gets the item it is searching for on the first iteration.

public static int binarySearch(int arr[], int first, int last, int key){

if (last>=first){
int mid = first + (last - first)/2;
if (arr[mid] == key){
return mid; }
if (arr[mid] > key){
return binarySearch(arr, first, mid-1, key);
}
else{
return binarySearch(arr, mid+1, last, key);
}
}
return -1;
}
